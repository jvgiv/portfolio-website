Node.js experience

Created a complete backend from table design upwards. Was responsible for designing the backend for a team build project. Worked through with migrations and seeding to create the initial tables. The tables are based off of one original user table that has a child table which also has two children itself. CRUB operation routes and functions were designed to make the data easily accessible to the front end developer. Authentication methods were used here. Middleware was created that gives the user a token that is comprised of an id, a secret, and a role.This site has been deployed to Heroku. The instructions for all CRUD operations have been posted on Postman.

Simpsons Says

I built an entire front end application to handle all of the CRUD operations from a server that someone on my team built for the same project. This was a coordinated effort between about 6 other people, all doing different aspects of the job. This is a fully functional site that allows you to register, log in, search for Simpsons quotes, save said quotes, and even generate quotes from select characters that were programmed through an algorithm to be things that the characters might say. This app was mainly built using Redux. This site has been deployed to Netlify.

Secure Login using Bcryptjs with React Interface

I designed this API to allow for logins to occur. Only after logging in can you see the users page. There is a logout button as well. This login system requires tokens which must match the server side token in order to be 'logged in'. This site also required the use of knex with Seeds and Migrations to create the data tables.

API Creation with React Interface

I designed this API to hold a few bits of information. Shortly after it's completion, I created a React interface to perform all CRUD operations. Add, edit, and delete are all fully functional
